Fast figures from your OMERO images

Smart figures with metadata

OMERO is a server platform for managing biological images. See the Open Microscopy Site for more info. OMERO.figure combines OMERO's powerful image rendering and metadata to provide a tool for rapid figure creation. Each panel of the figure becomes a multi-dimensional image viewer, allowing you to zoom and pan, adjust rendering settings and even scroll through Z and Time. Figures can be exported as TIFF images or PDF documents, so you can move seamlessly to other editing tools such as Adobe Illustrator.

Shortcuts for common tasks

Many tasks of figure creation are very time consuming when done manually. OMERO.figure provides tools to get the job done quickly. All of these features work with multiple selected panels at once, making it easy to keep similar panels in sync.

  • Align to grid: Snap panels to perfectly-spaced rows and columns.
  • Adjust rendering settings: Toggle channels, change colours and adjust levels.
  • Clever labels: Labels are bound to the chosen spot on each panel and can be auto-generated from image metadata.
  • Copy and paste: Duplicate rows or columns instantly, aligned perfectly.
  • Scroll Z and Time: Move image panels through Z and Time dimensions.
  • Zoom, Pan and Rotate: Crop your images simply by zooming, panning and rotating the viewport without ever losing the full image.
  • Crop to ROI: Use Regions of Interest from the OMERO server to crop images.
  • Accurate scalebars: Use the pixel size metadata from OMERO to add 'live' scalebars that update as you zoom.
  • Full undo and redo: For extra convenience.
  • Set 'dpi': You can control the resolution of images embedded in exported PDF pages.
  • Multi-page figures: Create, edit and view several figure pages within one OMERO.figure file at once.
  • Export TIFF or PNG: Export as multi-page PNGs or TIFF images (at 300 dpi) suitable for journal submission or further editing.
  • Export with panel images: Optionally export all panels as separate TIFFs to validate figure building steps or for use elsewhere.

Power of the web

You can create beautiful figures in your browser, then share them with others. Depending on your OMERO permissions settings, you can even make them public, so everyone can explore your figures interactively.

Install instructions

  • Download the release from the top of this page, unzip and rename the unzipped directory to 'figure'.
  • Place 'figure' within a directory that is on your PYTHONPATH, or in the OMERO lib/python/omeroweb/ directory. NB: for OMERO 4.4.9 and earlier, you will have to use lib/python/omeroweb/ (other locations not supported).
  • Add the app to OMERO.web: $ bin/omero config append omero.web.apps '"figure"'   # NB: double quotes NB: On Windows, you'll need to escape the inner double quotes: $ bin/omero config append omero.web.apps "\"figure\"" NB: for OMERO 4.4.x you'll need to 'set' the whole list (append not supported): $ bin/omero config set omero.web.apps '["figure"]'   # OMERO 4.4 only
  • Display a link to 'Figure' at the top of the webclient: $ bin/omero config append omero.web.ui.top_links '["Figure", "figure_index", {"title": "Open Figure in new tab", "target": "figure"}]'
  • Restart web: $ bin/omero web stop
    $ bin/omero web start

To enable PDF generation:

  • Install reportlab PDF python package.
    In the simplest case: $ pip install reportlab OR $ easy_install reportlab
    NB: For python 2.6, you'll need Reportlab 2.7: $ pip install reportlab==2.7
  • Optional: Figure legends can be formatted using Markdown syntax. To see this correctly in the exported PDF info page, we need Python-markdown. In the simplest case: $ pip install markdown OR $ easy_install markdown
  • Upload figure export script (need to be Admin): $ cd figure/scripts       # NB: important to be in this directory
    $ path/to/bin/omero script upload omero/figure_scripts/Figure_To_Pdf.py --official

Update instructions

  • To update an existing install of OMERO.figure to the latest release, download it, unzip and replace the contents of the current /figure directory with the contents of the unzipped directory.
  • NB: If you are upgrading from OMERO.figure 1.0.0 to 1.1.0 or later, note that we now use Python-Markdown (optional, see install instructions above).
  • Replace PDF generation script (need to be Admin): $ cd figure/scripts       # NB: important to be in this directory
    # List scripts to get the ID of omero/figure_scripts/Figure_To_Pdf.py
    $ path/to/bin/omero script list
    $ path/to/bin/omero script replace <scriptId> omero/figure_scripts/Figure_To_Pdf.py
  • If upgrading from OMERO.figure 1.0.0 to 1.1.0 or later, you can update the webclient link to figure, so that it automatically opens the figure app in a new tab. Simply remove the old link and add back the new one... # In case you need to list current links...
    $ bin/omero config get omero.web.ui.top_links

    # Remove the figure link...
    $ bin/omero config remove omero.web.ui.top_links '["Figure", "figure_index"]'

    # Add back new link as described in the install instructions above     NB: If your OMERO.web version is earlier than 5.1.0 then this change will be ignored until you upgrade.
  • Restart web: $ bin/omero web stop
    $ bin/omero web start
